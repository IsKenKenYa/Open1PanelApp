# 仪表盘模块FAQ

## 常见问题

### 1. 仪表盘数据多久刷新一次？

**默认配置**:
- 概览数据：每60秒刷新
- 资源监控：每5秒刷新
- 通知列表：每30秒刷新

**自定义**: 可在设置中调整刷新间隔

### 2. 为什么CPU使用率显示异常？

**可能原因**:
- 系统负载瞬时波动
- 数据采集延迟
- 多核CPU显示问题

**解决方案**:
1. 等待数据自动刷新
2. 手动刷新仪表盘
3. 检查系统监控服务状态

### 3. 如何自定义仪表盘布局？

**当前状态**: 暂不支持自定义布局

**未来规划**: 后续版本将支持拖拽式布局自定义

### 4. 内存使用率一直很高怎么办？

**排查步骤**:
1. 查看进程内存占用排行
2. 检查是否有内存泄漏
3. 清理系统缓存
4. 考虑增加物理内存

### 5. 磁盘空间不足告警如何处理？

**处理方法**:
1. 查看磁盘使用详情
2. 清理日志文件
3. 清理软件包缓存
4. 删除不需要的文件
5. 扩展磁盘容量

### 6. 快捷操作入口可以自定义吗？

**当前状态**: 使用系统默认配置

**未来规划**: 后续版本将支持自定义快捷操作

### 7. 通知中心的通知会保留多久？

**保留策略**:
- 未读通知：永久保留
- 已读通知：保留30天
- 告警通知：保留90天

### 8. 为什么网络流量图表显示为空？

**可能原因**:
- 网络接口未配置
- 数据采集服务未启动
- 权限不足

**解决方案**:
1. 检查网络接口状态
2. 重启监控服务
3. 检查应用权限

### 9. 如何查看历史监控数据？

**查看方法**:
1. 点击对应监控卡片
2. 进入详细监控页面
3. 选择时间范围
4. 查看历史趋势图

### 10. 仪表盘加载很慢怎么办？

**排查步骤**:
1. 检查网络连接
2. 清除应用缓存
3. 检查服务器负载
4. 减少刷新频率

## 故障排查指南

### 数据加载问题
```
错误: 仪表盘数据加载失败
排查:
1. 检查API连接状态
2. 验证用户权限
3. 查看服务端日志
4. 尝试重新登录
```

### 图表显示问题
```
错误: 图表不显示或显示异常
排查:
1. 检查数据是否正常返回
2. 清除浏览器缓存
3. 检查图表组件状态
4. 查看控制台错误
```

### 实时更新问题
```
错误: 数据不更新
排查:
1. 检查定时器是否运行
2. 验证网络连接
3. 检查API响应
4. 重启应用
```

## 最佳实践
